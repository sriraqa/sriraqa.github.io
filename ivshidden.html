Interac Verification Service

Project that I worked on while I was an intern on the Interac Verified team. Goal of this project was to improve user conversions on the Interac Verification Service.


Role: Product Designer


Team: 2 Designers, 1 Product Manager


Date: Nov 2023 (4 Weeks)


Skills: UI/UX Design (Figma), UX Research (UserTesting)


Click on the images for a zoomed in view







Introduction

The Interac verification service is a solution designed to help verify someone's identity online or in-person. Interac verification service allows them to access digital services by establishing a connection with their financial institution using bank login information.


This service is often used by Canadians as an option to sign into the Canada Revenue Agency (CRA) without needing to create a CRA account. It is also used on online retailer websites, such as Costco, to verify the user's identity before viewing sensitive personal information like a Costco membership. These are the landing screens that user would have seen as of November 2023:




While I was working as a product design intern at Interac, I was tasked with designing and testing some proposed changes from prior research. At the time, around 20% of users would not proceed with the verification service after opening the landing page. It was found that users would choose to not proceed because they felt overwhelmed by the information presented and/or unsure about their security when entering their banking details. So, we asked:

How might we improve the current design of the landing page to increase the number of user conversions through cleaner design and improved copy.

For this project, I worked with a senior UX designer to gain supporting feedback on my designs. I was also directed by a product manager for timelines and additional feedback.


Initial Design

After defining the goal, I wanted to create an initial web and mobile design that was based on prior research conducted by the product manager and my knowledge of modern app layouts. To do this, I needed to identify the elements in the design that were important and the potential weak points that may have been leading users to not proceed.


Strengths

  • Reassuring security and legal information is stated explicitly in many ways.
  • Multiple options for users that may prefer using the mobile app over the web version.

Weaknesses

  • The design has a lack of focus because of the amount of information.
  • Much of the copy is overly detailed.
  • It is unclear how complicated this process could be and might seem more complex than it is.
  • There is a lack of clear visual hierarchy that might prevent the user from intuitively knowing what to do.

By outlining these strengths and weaknesses of the current design, I created new web and mobile designs and asked for feedback on them from my team. For both, the main improvements were the addition of a progress stepper and an updated content layout.



For the web content layout, options were designed to afford a button shape by adding a rectangular border and shadow around each logo. Moreover, the existing mobile app was being promoted at the top as a more convenient option for the user. There is also a search bar for when more options are added.



For the mobile content layout, options were designed to be a scrollable list that can allow for more options in the future. Moreover, the existing mobile app is promoted as a fixed pop up that would open the app from the user's phone.


After presenting this design, I received feedback from the product manager and other designer to make improvements based on certain constraints. Specifically, the team did not want the mobile app to be promoted because it would not be a more convenient option than the web version for most users. Moreover, there were improvements made to increase the contrast on the the progress bar and button borders, so that the design could fit accessibility guidelines for WCAG 2.1.




User Research

After receiving feedback, I iterated further with the other designer and created two designs for the web that we wanted to AB test with potential users. The goal of the AB test was to identify which grid layout (fully boxed vs. Individual boxes) and presentation of text will make users more inclined to use the product and feel more secure. On the right of each image, the hypotheses for each element that we were hoping to confirm or contradict through user testing are outlined.


1. Version A
Fully boxed option grid
  • Less overall visual clutter.
  • Potentially misleading because the logos are not clearly CTAs unless user is hovering on them.
  • Similar to the original grid design.
Callout section for security information
  • More noticeable by users with worries because of greater visual hierarchy.
  • Could cause additional visual noise or distraction.

2. Version B
Individually boxed option grid
  • The CTAs look more actionable/clickable.
  • Potential of having additional visual noise due to borders around the logo buttons.
  • New grid design.
Subtitle section for security information
  • Less visual noise or distraction.
  • Less noticeable by users because of lower visual hierarchy.

Findings

  • 4/10 mentioned not using the service if there are a large number of steps.
  • 7/10 testers said that they would use the product.
  • Testers that might not use it had concerns with the number of steps and potential company data breaches with the large number of parties involved.
  • Testers that would use it mentioned that they trusted the parties involved, whether that be Costco, Interac, or the bank that they use.
  • Some testers mentioned that there might be users that are more anxious about security that would want more FAQ, but it was good that it was also not overwhelming.
  • Users rated the layout of the individual boxes slightly higher than the fully boxed version.


The Solution

Based on these findings, I created the final design:



In this design, we settled on the individual grid option with the callout because it received an overall higher score for visual appeal to users. Moreover, many users mentioned confusion with the number of steps, so the progress stepper from the first iteration was added back. For more wary users, we added a “Learn more” link on the callout for more FAQ.

Outcomes and Lessons

Overall, I was able to learn a lot about different areas of design that had never been exposed to or conducted on my own. Through collaboration on the designs with a more experienced designer and product manager, I was able to make design decisions based on what I had learned theoretically in courses that also aligned with constraints that were specific to this task and company. The other designer was my main guide for outlining constraints that I was unaware of and helping me to problem solve through them. For example, there were many legal constraints that I was unaware about for the copy that prevented the use of certain words or phrases that initially seemed intuitive to me.


Moreover, this was my first self-directed user test, so I learned to conduct A/B tests and analyze the nuances from the user recordings on my own. I really learned to be a better problem solver that goes through an iterative process and a better researcher that can properly create tests and analyze the results.


For future projects, I would like to go through a more systematic approach to my research. Since there was an upcoming deadline, I felt pressure to complete the testing sooner than it should have been. Ideally, I would have liked to document further competitive research and heuristic analysis that could have been incorporated in the user tests to gain further insight on user familiarity and preference.
